Build me a full-stack SaaS web application called "TourVision".

TourVision is an AI-powered real estate tool that automatically 
converts any property listing URL (Zillow, Airbnb, Bayut, 
Property Finder) into a fully navigable, photorealistic 3D 
virtual tour — powered by World Labs Marble API. 
No video needed. No special equipment. Just a link.

The agent pastes a listing URL. TourVision scrapes all photos, 
identifies each room using AI, sends the best photo of each 
room to World Labs Marble API which generates a navigable 
3D world per room, stitches all rooms into one seamless 
property tour, and returns a single shareable link the agent 
sends to buyers. Buyers open the link on any device, 
no download needed, and walk through the property in 3D.

---

TECH STACK — use exactly these, no substitutions:
- Framework: Next.js 14 with App Router
- Styling: Tailwind CSS + shadcn/ui components
- Animations: Framer Motion throughout
- Database: Supabase (PostgreSQL)
- Authentication: Supabase Auth 
  (email/password + Google OAuth)
- Payments: Stripe 
  (subscriptions + customer portal + webhooks)
- Background jobs: Inngest 
  (all async processing — never block the UI)
- File storage: Cloudflare R2
- Email: Resend
- SMS/WhatsApp: Twilio
- Deployment: Vercel
- All environment variables stored in .env.local

═══════════════════════════════════════
API ROUTES — build all of these:
═══════════════════════════════════════

POST /api/tours/create
Input: { listing_url: string }
Auth: required
Logic:
1. Check user tour limit for this month
2. If at limit: return 403 with 
   upgrade message
3. Create tour record in DB 
   with status 'pending'
4. Generate unique share_token
5. Trigger Inngest event: 
   'tour/process.started'
6. Return: { tour_id, share_token }

POST /api/tours/extract
(Called by Inngest background job)
Input: { tour_id, listing_url }
Logic:
1. Update tour status to 'extracting'
2. Call Apify API with listing URL
   Use Zillow scraper for zillow.com
   Use Airbnb scraper for airbnb.com
   Use generic scraper for others
3. Extract all photo URLs + 
   listing metadata (address, price, beds)
4. Update tour with listing metadata
5. For each photo:
   a. Download photo to Cloudflare R2
   b. Call Claude Vision API:
      Prompt: "Analyze this real estate photo.
      Return JSON only:
      {
        room_label: string 
        (Living Room/Kitchen/Master Bedroom/
        Bedroom/Bathroom/Balcony/Garden/
        Exterior/Other),
        quality_score: number 1-5
        (1=blurry/dark, 5=sharp/bright/wide),
        is_interior: boolean,
        is_best_angle: boolean
      }"
   c. Save to tour_photos table
6. Mark best photo per room 
   (highest quality_score)
   Set is_best_for_room = true
7. Update tour: 
   total_photos_extracted, 
   rooms_detected, 
   status = 'generating'
8. Trigger next Inngest step

POST /api/tours/generate-3d
(Called by Inngest background job)
Input: { tour_id }
Logic:
1. Get all best-for-room photos 
   for this tour
2. For each room (in parallel, 
   max 3 concurrent):
   a. Get photo from Cloudflare R2
   b. Upload to World Labs 
      signed upload URL
   c. Call World Labs API:
      POST to world generation endpoint
      Input: image asset ID
      Model: marble-1.1 (standard)
      Poll every 30 seconds until 
      status = 'completed'
   d. Extract world_marble_url 
      from response
   e. Save marble_world_id 
      to tour_photos record
   f. Update tour current_stage: 
      "Generating 3D world for 
      [Room Name]... ([N] of [N] complete)"
3. Save all marble_world_ids 
   to tours.marble_world_ids jsonb
4. Calculate confidence_score:
   (confirmed real photo angles / 
   total angles) × 100
5. Update tour status = 'ready'
6. Increment user tours_this_month + 
   total_tours
7. Trigger: send ready email via Resend
8. Return all world embed URLs

GET /api/tours/status/[tour_id]
Auth: required (tour owner only)
Returns:
{
  status: string,
  current_stage: string,
  rooms_completed: number,
  rooms_total: number,
  estimated_minutes: number,
  confidence_score: number
}
Used for polling every 15 seconds 
from processing screen

GET /api/tours/[share_token]
Auth: NOT required (public)
Returns tour data for viewer page:
{
  address: string,
  platform: string,
  marble_world_ids: array,
  rooms: array of room names,
  confidence_score: number,
  real_angles: number,
  ai_high_angles: number,
  ai_low_angles: number,
  is_watermarked: boolean,
  agent_name: string,
  agent_logo: string (if white-label)
}
Also: increment tour view count,
log to tour_views table 
with IP, country (use IP geolocation), 
device type

POST /api/tours/[tour_id]/lead
Auth: NOT required (public)
Input: { name, email, phone, message }
Logic:
1. Save to buyer_leads table
2. Send email to agent via Resend:
   Subject: "New buyer lead from your 
   TourVision tour"
   Body: buyer contact details + 
   which tour they came from
3. Return: { success: true }

POST /api/verification/send
Auth: required
Input: { tour_id, seller_phone }
Logic:
1. Create verification_request record
2. Send WhatsApp via Twilio:
   To: +[seller_phone]
   Message: "Hi! [Agent Name] has 
   created a 3D virtual tour of your 
   property using TourVision AI. 
   Some viewing angles were 
   AI-estimated and would benefit 
   from real photos to improve 
   accuracy for buyers. 
   Please take 2-3 photos of: 
   [list of low-confidence rooms] 
   and reply to this message 
   with the photos. 
   Thank you! 
   View the current tour: 
   tourvision.app/tour/[token]"
3. Return: { success: true }

POST /api/angles/flag
Auth: NOT required
Rate limit: 3 per IP per tour
Input: { tour_id, angle_id, reason }
Logic:
1. Save to angle_flags
2. Count flags for this angle
3. If count >= 3: 
   notify tour owner by email
4. Return: { success: true }

World Labs API:
Base URL: https://api.worldlabs.ai
Auth: Bearer token (WORLD_LABS_API_KEY)
Flow:
1. POST /v1/assets/upload-url 
   → get signed upload URL
2. PUT to signed URL with image binary
3. POST /v1/worlds 
   → { asset_id, model: "marble-1.1" }
   → returns { world_id, status }
4. GET /v1/worlds/[world_id] 
   every 30 seconds until 
   status = "completed"
5. Extract world_marble_url 
   from response for embedding

Apify API:
Base URL: https://api.apify.com/v2
Key actors:
- Zillow: maxim/zillow-scraper
- Airbnb: apify/airbnb-scraper  
- Generic: apify/web-scraper
Call: POST /acts/[actor]/runs
Poll: GET /acts/[actor]/runs/[runId] 
until status = SUCCEEDED
Extract: dataset items (photo URLs, 
listing details)

Claude Vision API (Anthropic):
Model: claude-opus-4-5
Use for room labeling + quality scoring
Send image as base64 in message content
Parse JSON response strictly
Fallback if parsing fails: 
label = "Room", quality = 3
